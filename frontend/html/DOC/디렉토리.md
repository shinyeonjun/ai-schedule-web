# MUFI Frontend HTML 디렉토리 구조

## 📁 전체 구조
```
frontend/html/
├── DOC/                          # 📋 문서 폴더
│   └── 디렉토리.md              # 현재 파일 - HTML 구조 문서
├── dashboard/                     # 🏗️ 대시보드 템플릿 폴더
│   ├── main.html                 # 📄 메인 레이아웃 템플릿 (헤더, 사이드바, 콘텐츠 영역)
│   ├── sidebar.html              # 📋 사이드바 컴포넌트 (네비게이션 메뉴)
│   ├── header.html               # 🎯 헤더 컴포넌트 (알림, 사용자 프로필)
│   ├── modals.html               # 🪟 모달 컴포넌트 (로딩, 공유, 이메일 모달)
│   └── sections/                 # 📑 섹션별 컴포넌트
│       ├── analysis.html         # 📞 통화 분석 섹션 (파일 업로드, 분석 결과)
│       ├── schedules.html        # 📅 일정 관리 섹션 (일정 목록, ICS 다운로드)
│       └── members.html          # 👥 인원 관리 섹션 (사용자 목록, 연락처)
├── login.html                    # 🔐 로그인 페이지 (Google OAuth)
├── dashboard.html                # 🎯 최종 빌드된 대시보드 (템플릿 조합 결과)
├── dashboard_before.html         # 📜 원본 대시보드 (참고용)
└── build-dashboard.js            # 🔨 빌드 스크립트 (템플릿을 조합하여 dashboard.html 생성)
```

## 📋 파일별 상세 설명

### 🏗️ 템플릿 파일들
- **main.html**: 대시보드의 기본 레이아웃 구조. `{{SIDEBAR}}`, `{{HEADER}}`, `{{CONTENT}}`, `{{MODALS}}` 플레이스홀더 포함
- **sidebar.html**: 좌측 네비게이션 바. 통화 분석, 일정 관리, 인원 관리 메뉴
- **header.html**: 상단 헤더. 알림 시스템, 사용자 프로필, 로그아웃 버튼
- **modals.html**: 공통 모달 컴포넌트들 (로딩 오버레이, 공유 모달, 이메일 모달)

### 📑 섹션별 컴포넌트
- **analysis.html**: 통화 분석 기능. 파일 업로드 영역, 분석 결과 표시, 일정 추출
- **schedules.html**: 일정 관리 기능. 생성된 일정 목록, ICS 파일 다운로드
- **members.html**: 인원 관리 기능. 사용자 목록, 연락처 관리

### 🎯 최종 파일들
- **login.html**: Google OAuth 로그인 페이지. 브랜딩 섹션과 로그인 폼
- **dashboard.html**: 빌드 스크립트로 생성된 최종 대시보드 (모든 템플릿 조합)
- **dashboard_before.html**: 원본 대시보드 파일 (참고용, 템플릿 분리 전 버전)

### 🔨 빌드 시스템
- **build-dashboard.js**: Node.js 빌드 스크립트. 템플릿들을 조합하여 최종 dashboard.html 생성

## 🔄 빌드 프로세스
1. `main.html` 템플릿 로드
2. `sidebar.html`, `header.html` 컴포넌트 로드
3. `sections/` 폴더의 모든 섹션 파일 로드
4. `modals.html` 모달 컴포넌트 로드
5. 플레이스홀더 치환하여 최종 `dashboard.html` 생성

## 📝 사용법
```bash
# 빌드 실행
cd frontend/html
node build-dashboard.js
```

## 🎯 특징
- **모듈화**: 각 기능별로 독립적인 HTML 파일
- **재사용성**: 컴포넌트 단위로 분리하여 유지보수 용이
- **빌드 시스템**: 템플릿 조합으로 최종 파일 자동 생성
- **일관성**: 모든 페이지가 동일한 구조와 스타일 적용
